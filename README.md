# 知乎圈子监控服务
## 项目简介
基于 FastAPI 实现的知乎圈子自动化监控服务，支持圈子管理、帖子数据抓取、AI 内容分析、SSE 流式实时输出，前端可视化控制台展示监控日志与结构化数据。
## 技术栈
+ 后端：Python + FastAPI
+ 通信：SSE 长连接流式推送
+ 跨域：CORS 全局放行
+ 前端：原生 HTML + JS + 内置样式
## 项目功能
1. 圈子管理
+ 获取已配置圈子列表
+ 新增自定义监控圈子
2. 任务监控
+ 支持指定单个圈子 / 全量圈子监控
+ SSE 实时流式输出处理日志、结构化结果
3. 数据处理
+ 知乎圈子帖子接口数据拉取
+ 自定义数据解析提取
+ AI 内容流式分析输出
## 接口说明
### 基础服务地址
> http://127.0.0.1:8000
1. 获取圈子列表
请求方式：GET
接口地址：/api/circle/list
描述：获取所有配置的圈子名称与 ID 映射
响应格式：
json
{
  "code": 200,
  "msg": "success",
  "data": {
    "circle_list": {}
  }
}
2. 添加圈子
请求方式：POST
接口地址：/api/circle/add
请求体：
json
{
  "name": "圈子名称",
  "circle_id": "圈子唯一ID"
}
说明：圈子 ID 重复会返回 400 错误
3. 启动监控任务（SSE 流式）
请求方式：POST
接口地址：/api/task/run
请求体（可选）：
json
{
  "circle_id": "指定圈子ID"
}
不传参：监控所有圈子
传参：仅监控指定单个圈子
响应类型：text/event-stream 长连接，持续流式推送日志与结果

## 运行部署
1. 安装依赖
bash
运行
uv sync 
2. 启动后端服务
bash
运行
python main.py
服务默认运行在：http://0.0.0.0:8000
## 特性说明
全局跨域放行，支持前端直接对接
队列中转消息，SSE 统一推送到前端
前端自动格式化 JSON 输出，保留排版缩进
页面刷新自动断连，避免后端任务堆积
## 注意事项
手动刷新前端页面会断开 SSE 长连接，属于浏览器天然特性
新增圈子需填写合法圈子名称和唯一圈子 ID
依赖的配置、接口、工具模块需保证路径导入正常
